Interface Audit — Landing Page

Visibility of System Status

The landing page functions as a central hub for the sketchbook. While the header clearly identifies the site, the individual system states remain opaque. A primary issue is the lack of visual indicators for "visited" links. When a user explores a sketch and returns, the button appears unchanged. This forces the user to mentally track their progress through 14 different interactive points, violating the principle of "Recognition rather than Recall." Without a color shift or progress marker, the system fails to communicate which tasks have been completed.

Affordances & Signifiers

Labeling buttons "Sketch 1" through "Sketch 9" creates a "blind-click" scenario where the user cannot predict the interaction style or theme before clicking. Furthermore, the grouping under "Micro Project" uses the same visual styling for analytical documents (Reflection/Audit) and the interactive project (FINAL), failing to signify the difference between a reading task and a play task.

Navigation & Mapping

The system currently employs a rigid Hub-and-Spoke mapping model. This creates a high interaction cost; if a user is in Sketch 1 and wishes to view Sketch 2, they must click the browser's back button to return to the hub first. There is no lateral mapping to allow users to move sequentially through the work. This "back-arrow dependency" creates a disjointed experience where the landing page acts as a mandatory roadblock rather than a fluid directory.

Constraints & Assumptions

The interface assumes a high level of user patience and a linear browsing intent that isn't structurally enforced. It hides technical constraints. The design assumes that the user will intuitively understand that "Ideas" and "Step One" should precede "FINAL," yet provides no visual arrows or sequence lines to guide that behavior.

Heuristic Violations

  • Recognition rather than Recall: Numbered buttons force users to memorize content locations. Fix: Add hover-previews or descriptive tags.
  • User Control and Freedom: Users lack an "Emergency Exit" within sketches to return home quickly or move to the next item. Fix: Implement a site-wide Navbar.

Structural Revision Proposal

  • Global Navigation Bar: Implement a persistent navbar across all pages (Hub, Sketches, and Projects). This allows for "Lateral Navigation," enabling users to jump between any two pages without returning to the home screen.
  • Stateful Buttons: Integrate CSS :visited logic or JS-based toggle states to change the button color once a user has viewed a specific sketch, providing immediate feedback on completed tasks.
  • Visual Pathing: Link the Micro Project steps with a literal line or "Step 1, 2, 3" markers to visually guide the user through the project's evolution chronologically.
