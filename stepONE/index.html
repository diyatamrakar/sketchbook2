Micro-Project — Progress #1

1. Concept Summary

My project is a digital "skipping pond" where the user clicks to throw a stone across the water. The user controls the starting position of the throw with their mouse. As time passes, the stone automatically bounces across the screen, creating ripples and sound-words ("PLIP", "THUD", "SPLASH") until it loses energy and sinks.

2. Primary Variable

let bounceCount = 0;

What changes this variable? The variable increases by 1 every time a ripple finishes growing and the stone "jumps" to its next position.

What does it control? It controls the behavior of the stone. It tells the computer which word to display and decides when the stone has bounced too many times and needs to sink.

3. Base Build (Step One)

The current version uses the bounceCount variable and a mouse click to move between two distinct states:

  • State 1 (Idle): The screen is a still, blue pond with text that says "Click to skip a stone." The system is waiting for user input.
  • State 2 (Active): Once clicked, the stone enters a movement loop. It triggers three ripples in a sequence, each with a different text label, and slows down (friction) after every bounce.

4. Structural Growth

In the coming weeks, I will make the pond more realistic by adding a gravity variable so the stone travels in a curved "hop" (parabolic arc) instead of a flat line. I will also implement Object-Oriented Programming (OOP) so the user can throw multiple stones at the same time. Finally, I will add a "Choppy Water" threshold: if there are too many ripples on screen at once, the water becomes turbulent, causing new stones to lose speed faster and sink earlier.
