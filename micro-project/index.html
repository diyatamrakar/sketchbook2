Micro-Project — Progress Report #2

1. Current Build

The current system simulates a stone skipping across a pond using a multi-state logic system. My main variable is bounceCount, which tracks the stone's progression through three distinct phases: "PLIP," "THUD," and "SPLASH." When the user interacts by clicking, the system locks the ripple coordinates to the point of impactX/Y while the pebble continues forward. The pebble automatically disappears (sinks) when it triggers the final "SPLASH" threshold.

2. Where Is the Pressure?

Tension in this system is threshold-based. The movement of the stone is gated by the rippleSize variable; the system is essentially held in a "waiting state" until the ripple reaches its maximum expansion of 150 pixels. This creates a rhythmic delay that mimics physical water resistance and surface tension, forcing the user to observe the full animation cycle before the next state is allowed to trigger.

3. One Structural Addition: Threshold-Based Behavior

I plan on implementing a Multi-Impact Tracking System. By giving each point of impact its own independent life cycle, ripples no longer disappear when a new bounce occurs. Instead, waves from the initial 'PLIP' continue to expand and overlap with the subsequent 'THUD' and 'SPLASH' impacts. This creates a layered interference pattern that realistically simulates how multiple wave fronts collide on a water surface.
